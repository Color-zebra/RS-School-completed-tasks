FSD Architecture
An architecture is a set of rules and conventions for organizing code. The main goal is to make the project understandable and structured, especially in the context of regular changes and additions.
  • Exclusively for the frontend.
  • May be overkill for small projects.
  • May not be suitable for VERY large applications.
  • Suitable for any framework or language.
A layer is the first level of an organizational hierarchy. Their goal is to separate the code based on the degree of responsibility and dependencies on other modules.
Slices are the second level of the organizational hierarchy. Their main purpose is to group code by its value to the application.
A public API is a public interface that provides a single point of access to all the capabilities of a module and determines how this module will interact with the outside world.
  • Protects the internal structure of modules from uncontrolled change.
  • Helps to encapsulate modules.
  • Allows you to easily define module behaviors based on API methods.
Segments are the third and final level of the organizational hierarchy. Their task is to group the code according to its technical nature.

Should be

  • Layering.
  • Public API for slices.
  • Top-down interaction.

Optional

  • Implementation of all layers.
  • Implementation of slice segmentation.
